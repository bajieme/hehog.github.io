# 1、创建一个工厂，生成PayContext对象。

public class PayFactory {

    /**
     * 获取PayContext
     *
     * @param template 、
     * @return 、
     */
    public PayContext getPayContext(String template) {
        if (StringUtils.isEmpty(template)) {
            return null;
        }
        switch (template) {
            case ALI_PAY:
                return new PayContext(new AliPayStrategy());
            case WEIXIN_PAY:
                return new PayContext(new WeixinPayStrategy());
            default:
        }
        return null;
    }
}

# 使用该工厂，通过传递支付类型类型信息来获取PayContext。

public class MainCoreFactoryTest {

    public static void main(String[] args) {
        int amount = 199;
        designPay(ALI_PAY,amount);
        designPay(WEIXIN_PAY,amount);
    }

    /**
     * 支付
     *
     * @param payType 支付类型
     * @param amount  支付金额
     */
    public static void designPay(String payType, int amount) {
        PayFactory payFactory = new PayFactory();
        PayContext payTemplate = payFactory.getPayContext(payType);
        payTemplate.pay(amount);
    }
}

# 总结

通过以上模板、策略、工厂三种设计模式就完成了队if判断代码的改造，只需要通过前端传过来的payType就可以执行具体的支付流程。而且我们遵循了设计模式的开闭原则，开：可以添加新的支付方式，闭：不能修改支付的流程。我们在开发的过程中要多思考，有时候设计模式能给我们带来很大的帮助，但是不能过度设计，有的地方用if判断就结束了，但是为了强行使用设计模式而加入设计模式也是不合理的，像上面的策略模式也是可以省略的，我是为了介绍策略模式才引入的。应该有很多同学已经看出来，其实只要模板模式+工厂模式就可以完成上面的代码改造。下一篇介绍下抽象工厂
